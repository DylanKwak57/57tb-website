## 1. Overview

### 1.1 Purpose

현재 57tb.art 사이트(Notion/OOpy 기반)를 Next.js 기반 프리미엄 미용실 웹사이트로 전면 리뉴얼하여:
- 고객이 사이트를 보는 것만으로 "여기서 머리하고 싶다"는 감정을 느끼게 함
- 방콕 한국 미용실 중 압도적으로 차별화된 웹 경험 제공
- 온라인 예약/상담 전환율 극대화

### 1.2 Background

**현재 사이트 (57tb.art) 분석:**
- Notion CMS + OOpy 렌더러 기반 → 커스텀 한계, 성능 제한
- 다크 테마, 기본적인 정보 제공 (가격표, 갤러리, 위치, 리뷰)
- 온라인 예약 시스템 부재 (전화/LINE만 가능) 우리 회사 온라인 시스템은 이미 자체 프로그램으로 구현되어져 있으며 라인 리치메뉴를 통해 바로 가능함.https://57totalbeauty.net/booking/home-8D50D7DB-1BF7-4403-A0F6-CA32095E5E64
- 다국어 지원 미흡 (태국어 중심, 한국어 없음)

**경쟁사 분석:**
- The Line Salon: Wix 기반, 기본적 정보만 제공
- Senpom Salon: 예약 없음, 갤러리 중심
- Green Pastures by Hajin: Instagram 의존
- **핵심 인사이트**: 방콕 한국 미용실 중 제대로 된 웹사이트가 거의 없음 → 선점 기회

**2026 글로벌 살롱 웹사이트 트렌드:**
- 몰입형 풀스크린 비주얼 (패럴랙스, 시네마틱 영상)
- AI 기반 헤어스타일 가상 체험 (Virtual Try-on)
- 원클릭 온라인 예약 (70%+ 고객이 온라인 예약 선호)
- LINE/SNS 완전 통합 (태국 시장 필수)

### 1.3 Related Documents

- 현재 사이트: https://www.57tb.art/
- 경쟁사 참고: The Line Salon, Senpom Salon, Salon 64
- 트렌드 참고: prempage.com, colorlib.com, weblium.com 2026 살롱 웹사이트 가이드

## 6. Architecture Considerations

### 6.1 Project Level Selection

| Level | Characteristics | Recommended For | Selected |
|-------|-----------------|-----------------|:--------:|
| **Starter** | Simple structure (`components/`, `lib/`, `types/`) | Static sites, portfolios, landing pages | **O** |
| Dynamic | Feature-based modules, BaaS integration | Web apps with backend, SaaS MVPs | |
| Enterprise | Strict layer separation, DI, microservices | High-traffic systems | |

> **Starter** 선택 이유: Phase 1은 정적 웹사이트(예약 시스템 없음), 백엔드 불필요, 빠른 구현 우선

### 6.2 Key Architectural Decisions

| Decision | Options | Selected | Rationale |
|----------|---------|----------|-----------|
| Framework | Next.js / Astro / React | **Next.js 14 (App Router)** | 이미지 최적화, SSG, i18n 지원, Dylan 기존 스택 |
| Styling | Tailwind / CSS Modules / styled-components | **Tailwind CSS v4** | 빠른 개발, 반응형 유틸리티, Dylan 기존 스택 |
| Animation | Framer Motion / GSAP / CSS | **Framer Motion** | React 친화적, 선언적 API, 패럴랙스 구현 |
| i18n | next-intl / next-i18next | **next-intl** | App Router 네이티브 지원, 간결한 API |
| Image | Next Image / Cloudinary | **Next Image + sharp** | 자동 WebP 변환, 빌트인 최적화 |
| Deployment | Vercel / GitHub Pages / Netlify | **GitHub Pages** (테스트) → Vercel (프로덕션) | GitHub MCP 연결됨, 테스트 편의성 |
| Font | Google Fonts / self-hosted | **Google Fonts (Noto Sans TH/KR + Montserrat)** | 다국어 폰트 커버리지 |

### 6.3 Clean Architecture Approach

```
Selected Level: Starter

Folder Structure:
┌──────────────────────────────────────────────┐
│ src/                                         │
│   app/                                       │
│     [locale]/                                │
│       page.tsx          ← 메인 랜딩페이지     │
│       services/page.tsx ← 서비스 & 가격       │
│       gallery/page.tsx  ← 스타일 갤러리       │
│       team/page.tsx     ← 스타일리스트 소개    │
│       location/page.tsx ← 지점 안내           │
│       layout.tsx        ← 공통 레이아웃        │
│     layout.tsx          ← 루트 레이아웃        │
│   components/                                │
│     ui/                 ← 공통 UI 컴포넌트     │
│     sections/           ← 페이지 섹션 컴포넌트  │
│     layout/             ← 헤더, 푸터, 네비     │
│   lib/                                       │
│     constants.ts        ← 상수 (가격, 정보)    │
│     utils.ts            ← 유틸리티 함수        │
│   messages/                                  │
│     th.json             ← 태국어              │
│     en.json             ← 영어               │
│     ko.json             ← 한국어              │
│   types/                                     │
│     index.ts            ← 타입 정의           │
│ public/                                      │
│   images/               ← 이미지 에셋         │
│   videos/               ← 비디오 에셋         │
└──────────────────────────────────────────────┘
```

## 8. Design Direction

### 8.1 Visual Concept

**컨셉: "Cinematic Luxury meets K-Beauty"**

- 풀스크린 시네마틱 히어로 (실제 시술 장면 영상/이미지)
- 블랙 & 골드 프리미엄 색상 팔레트
- 대담한 타이포그래피 + 여백 활용
- 부드러운 스크롤 애니메이션 (패럴랙스, 페이드인)
- 스타일별 갤러리 필터로 시술 결과 탐색
